2.px、em、rem、%、vw、vh、vm等单位有什么区别？

  • px : 像素单位
  • em : 相对单位, 参考物是 父元素 的font-size，具有继承的特点。如果自身定义了font-size按自身来计算（浏览器默认字体是16px），整个页面内1em不是一个固定的值
  • rem : 相对单位, css3新单位，相对于 根元素 html（网页）的font-size，不会像em那样，依赖于父元素的字体大小，而造成混乱
  • % : 相对单位, 一般情况下是相对于父元素的
  • vw : css3 新单位，viewpoint width 的缩写，视窗宽度，1vw 等于视窗宽度的1% , 例子：浏览器宽度1200px, 1 vw = 1200px/100 = 12 px
  • vh : 和 vw 一个意思, viewpoint height的缩写, 视窗高度，1vh等于视窗高度的1%
  • vm : 喵喵喵, 好像不怎么用呢, 用到再说喽

5.什么时候用margin好，什么时候用padding好？

边界(margin)：元素周围生成额外的空白区。“空白区”通常是指其他元素不能出现且父元素背景可见的区域。——CSS权威指南
补白(padding)：补白位于元素框的边界与内容区之间。很自然，用于影响这个区域的属性是padding。——CSS权威指南

网上的总结 :

用Margin还是用Padding的区别, 留着以后有问题的时候看。

何时应当使用margin：
需要在border外侧添加空白时。
空白处不需要背景（色）时。
上下相连的两个盒子之间的空白，需要相互抵消时。如15px + 20px的margin，将得到20px的空白。

何时应当时用padding：
需要在border内测添加空白时。
空白处需要背景（色）时。
上下相连的两个盒子之间的空白，希望等于两者之和时。如15px + 20px的padding，将得到35px的空白。

6.手机分辨率和网页中的px是一回事吗？

手机中的分辨率分物理分辨率和逻辑分辨率，我们拿iPhone6来举例，它的逻辑分辨率是375 * 667，物理分辨率是 750 * 1334，逻辑分辨率应该就是和网页中的px一致了吧，逻辑分辨率是用 几个物理分辨率合并成1个逻辑分辨率。

比如Macbook pro 的 retina 显示技术，就是用这个原理实现的。用几个物理像素合并成一个实际的像素，然后就会非常清楚。（个人理解哈，有误的话欢迎指出，新人说错了请谅解谅解，一切都是个人暂时的理解）
